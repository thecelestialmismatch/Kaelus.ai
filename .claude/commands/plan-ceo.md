CEO-level review before building: $ARGUMENTS

Answer these before a single line of code:

## 1. CUSTOMER VALUE
- Which segment benefits? (Tech / Healthcare / Defense / Legal-Finance)
- Does this move toward $10K MRR or first 5 paying customers?
- Will a defense contractor pay more for this?

## 2. PRIORITY CHECK
- More important than getting the first paying customer?
- Does this unblock revenue or is it polish?
- Opportunity cost — what are we NOT doing?

## 3. SCOPE
- Minimal version that delivers core value?
- What can be cut without losing the value?
- Reversible or hard to undo?

## 4. RISKS
- Could this break existing customers?
- Any compliance/security implications?
- Does it touch the compliance engine? (needs compliance-specialist review)

## 5. DECISION
- **Build now** / **Build later** / **Don't build**
- If build now: hand to `/ralph` with this spec
